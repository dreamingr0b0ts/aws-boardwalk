Alpenglow Model Workbench

"Which model should we use?" is a procurement question, not a matter of taste. This workbench sends one prompt to four foundation models from four vendors — through the single Bedrock Converse API — and shows the answers side by side with measured latency, token counts, and computed cost. Every run lands in an audit ledger: who ran what, against which models, with which parameters.

    What this plank proves

    • Model selection as an engineering discipline: the same prompt, the same parameters, four vendors side by side — with latency, tokens, and cost measured, not estimated. The "cheapest capable model" conversation becomes evidence-based.
    • One API, any model: Bedrock's Converse API means the fan-out is a loop, not four integrations — swapping or adding a model is IAM + a config entry, which is exactly the exit-ramp a government buyer should demand from a GenAI vendor.
    • Auditable by construction: every run is logged with its parameters, token usage, and computed cost — the explainable audit trail responsible-AI policies require, demonstrated rather than promised.
    • Spend that cannot run away: Cognito gate with no self-signup and an unpublished credential, per-user and demo-wide daily run caps in DynamoDB, a hard output-token ceiling, and a 5 rps edge throttle. Idle cost is $0; worst-case leaked-credential day is a few dollars.
    • Grounding differences made visible: the refusal-test scenario shows which models admit "the excerpt doesn't say" and which ones confidently invent — the single most important behavior for civic AI.
