Alpenglow Land & Records Registry

Every submit→review→decide system a government buys rests on a relational core. This one is Aurora Serverless v2 PostgreSQL that genuinely scales to zero: 0 ACU floor, five-minute auto-pause, reached only through the RDS Data API — no NAT, no ALB, no open database port anywhere. When the demo window ends the whole cluster is torn down, and the evidence report below is what remains.

One command each way: make demo stands up the cluster, runs the checksummed migrations, generates ~20,000 rows in-engine, and writes fresh evidence; make teardown destroys every billing resource. While deployed, five idle minutes drop compute to 0 ACU — the first exhibit you run wakes it, and the page times the wake.

How a query gets here

Browsersame-origin /api/* — no CORS anywhere → CloudFront + Lambdacanned exhibits only — no user SQL exists → RDS Data APIHTTPS + IAM + Secrets Manager — no database socket → Aurora Serverless v2PostgreSQL 17 · 0→1 ACU · auto-pause 5 min → Evidence reportsurvives teardown

The cluster's security group has zero ingress rules — nothing can open a connection to it. The API connects as app_user, a Postgres role that can read the registry and write only the rollback sandbox; its DELETE and DROP attempts are one of the exhibits.

        What this plank proves
        
    
          
      
        • The relational fundamentals, live: foreign keys, CHECK constraints, and
      atomic transactions demonstrated by the engine's own error messages — not slideware claims.
      Every integrity exhibit runs inside a transaction that is always rolled back.
          • 
      
        • Scale-to-zero is real: min capacity 0 ACU with auto-pause. A quiet
      cluster bills no compute; the visible ~15-second wake when you run the first exhibit is the
      trade-off, measured honestly on this page.
          • 
      
        • No-socket access pattern: the RDS Data API means HTTPS + IAM +
      Secrets Manager instead of connection strings — a security group with zero ingress rules,
      no VPC-attached Lambdas, no NAT to pay for.
          • 
      
        • Least privilege below the IAM layer too: the public API's Postgres role
      can read the registry and write only the sandbox; the exhibit shows its DELETE and DROP dying
      in the engine.
          • 
      
        • Schema as code: ordered, checksummed migrations recorded in the database
      they shaped, rerunnable idempotently — and cost-honest architecture: RDS is banned always-on
      in this portfolio, so the whole cluster is deploy-demo-teardown with evidence persisted
      between windows.
