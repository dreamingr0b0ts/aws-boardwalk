Alpenglow Batch Works

Containers with no idle bill: press the button and a real Docker container is launched on ECS Fargate just for you — watch it provision, pull, run, and exit, log line by log line, then read the report it uploads to S3. Between runs, nothing is running anywhere.

Launch a container

Each launch is a genuine ecs:RunTask against a scale-to-zero Fargate cluster — 0.25 vCPU / 512 MiB, the smallest size Fargate sells (~$0.001 per run). One task runs at a time and there's a global daily cap, so if someone else's container is in flight you'll be handed theirs to watch instead.

The failing job hits a deliberate bad-input error and exits 1 — so you can see how a container failure actually surfaces: non-zero exit code, stopped reason, no artifact.

What this plank proves

    
    
      
  • Containers without the idle bill: no ECS service, no ALB, no EKS control
      plane ($73/mo) — tasks exist only while they run, so the environment idles at ~$0 yet is live
      right now. The classic fit for batch jobs, report generation, and queue workers.
    • 
      
  • A real image pipeline: source → CodeBuild → ECR with scan-on-push
      vulnerability findings surfaced above, and a lifecycle policy pruning old images.
    • 
      
  • Task IAM roles done right: the execution role (pull image, write
      logs) is separate from the task role, which can do exactly one thing — write the
      finished report under artifacts/. The report's footer is written by that role.
    • 
      
  • Observable by default: stdout → awslogs → CloudWatch, tailed live above;
      lifecycle state from DescribeTasks; final state persisted by an EventBridge
      task-state-change rule even if nobody's watching.
    • 
      
  • No NAT tax: tasks get a public IP in a public subnet with a no-ingress
      security group — the documented pattern for internet-reachable AWS APIs without a $32/mo NAT
      gateway.
    • 
      
  • Failure is part of the demo: the failing job shows exactly what operators
      get — exit code, stopped reason, and logs up to the fault.
